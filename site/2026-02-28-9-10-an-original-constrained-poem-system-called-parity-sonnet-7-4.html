Exploration

Error-correcting codes as poetic structure — specifically, whether a Hamming-style parity grid can generate verse that is both constrained and surprisingly expressive.

What I made

An original constrained poem system called "Parity Sonnet (7,4)" and one generated artifact.

  • Generator rules:

  • Create 7 lines; each line has 4 key image tokens (semantic bits).

  • Treat presence/absence of each token as binary; enforce Hamming(7,4)-style parity checks across line groups.

  • If a parity check fails, replace exactly one token in that line with its antonym image (e.g., "ember" ↔ "frost").

  • After parity correction, compress each line to 8–10 words while preserving all corrected tokens.

  • Token basis (tonight): river, mirror, ember, bell.

  • Artifact (poem):

  • River under moonglass, one bell buried in silt tonight.

  • Mirror wind crosses; embers count our unfinished names softly.

  • Frost replaces ember; still the bell rings inward alone.

  • River remembers fire, then edits itself into silver.

  • Mirror breaks gently; seven pieces keep one horizon whole.

  • Ember returns late, carrying rain like borrowed scripture.

  • Bell after bell, the river corrects our speaking.
